# ТЕСТ-ПЛАН
# Финансовый калькулятор v1.0

## 1. Объем тестирования

### Включается в тестирование:
- Все три модуля калькулятора (кредит, валюта, вклады)
- Функциональное тестирование
- Тестирование пользовательского интерфейса
- Тестирование обработки ошибок
- Тестирование граничных значений

### Исключается из тестирования:
- Интеграция с внешними системами
- Тестирование безопасности
- Тестирование на разных платформах
- Нагрузочное тестирование

## 2. План работ по этапам

### ЭТАП 1: Подготовка (3 дня)
- Настройка тестового окружения (.NET 6.0 SDK)
- Подготовка тестовых данных
- Создание тестовой документации
- Проверка готовности тестовых сценариев

### ЭТАП 2: Функциональное тестирование (5 дней)
- День 1-2: Тестирование кредитного калькулятора
- День 3: Тестирование конвертера валют
- День 4: Тестирование калькулятора вкладов
- День 5: Кросс-модульное тестирование

### ЭТАП 3: Регрессионное и приемочное тестирование (2 дня)
- Smoke-тестирование после исправлений
- Проверка критического функционала
- Приемочное тестирование

## 3. Критерии приемки

### Критерии приемки функционала:

**Для кредитного калькулятора:**
- Расчет аннуитетного платежа соответствует математической формуле
- Корректная обработка граничных значений
- Понятные сообщения об ошибках валидации

**Для конвертера валют:**
- Конвертация происходит по установленным курсам
- Корректная обработка всех пар валют
- Валидация кодов валют

**Для калькулятора вкладов:**
- Корректный расчет для вкладов с капитализацией и без
- Соответствие математическим формулам
- Правильное округление результатов

### Общие критерии:
- Отсутствие критических дефектов
- Стабильная работа приложения
- Соответствие функциональным требованиям
- Успешное прохождение smoke-теста
