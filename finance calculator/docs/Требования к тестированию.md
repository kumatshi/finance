# ТРЕБОВАНИЯ К ТЕСТИРОВАНИЮ
# Финансовый калькулятор v1.0

## 1. Функциональные требования

### Кредитный калькулятор
- Должен корректно рассчитывать аннуитетные платежи
- Должен обрабатывать валидные данные (сумма: 1000-10,000,000, срок: 1-360 месяцев, ставка: 0.1-99.9%)
- Должен валидировать некорректные входные данные
- Должен отображать ежемесячный платеж, общую сумму и переплату

### Конвертер валют
- Должен конвертировать между RUB, USD, EUR по фиксированным курсам
- Должен использовать актуальные курсы: USD→RUB=90.0, EUR→RUB=98.5, EUR→USD=1.09
- Должен обрабатывать конвертацию одинаковых валют
- Должен валидировать коды валют

### Калькулятор вкладов
- Должен рассчитывать доход для вкладов без капитализации
- Должен рассчитывать доход для вкладов с капитализацией
- Должен корректно применять математические формулы

### Общие функциональные требования
- Корректное отображение меню и навигация
- Обработка ошибок ввода с понятными сообщениями
- Восстановление после ошибок

## 2. Нефункциональные требования

### Производительность
- Время отклика на операции ≤ 2 секунды
- Потребление памяти ≤ 50 МБ

### Надежность
- Стабильная работа при корректных данных
- Корректная обработка исключительных ситуаций
- Отсутствие утечек памяти

### Удобство использования
- Интуитивно понятный текстовый интерфейс
- Русскоязычный интерфейс
- Понятные сообщения об ошибках

### Совместимость
- .NET 6.0 и выше
- Windows 10/11, Linux, macOS
- Консольные терминалы с поддержкой UTF-8

## 3. Риски

### Технические риски
- Некорректные математические формулы расчета
- Ошибки округления при финансовых расчетах
- Неточности при работе с плавающей точкой

### Процессные риски
- Недостаток времени на полное тестирование
- Неполные требования к граничным значениям

### Риски окружения
- Проблемы с локальной тестовой средой
- Различия в окружениях разработки и тестирования

## 4. Критерии начала и завершения тестирования

### Критерии начала тестирования
- Готовность тестового окружения
- Наличие стабильной сборки приложения
- Утвержденные тест-кейсы и чек-листы

### Критерии завершения тестирования
- Выполнено 100% тестов высокого приоритета
- Выполнено ≥90% всех запланированных тестов
- Исправлены все критические и серьезные дефекты
- Отсутствие блокирующих дефектов
- Успешное прохождение smoke-тестирования
